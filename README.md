# AWS Practice Plan

## 1. 학습 목표

본 학습의 최종 목표는 다음과 같습니다:

1. AWS의 핵심 개념 및 서비스 이해  
2. 기본적인 클라우드 아키텍처 설계 능력 확보  
3. 실제 서비스 배포 및 운영 가능 수준 도달  
4. AWS 자격증 취득 (Cloud Practitioner → Solutions Architect)

---

## 2. AWS 개념 정의

AWS는 다음과 같이 정의할 수 있습니다:

> 인터넷을 통해 서버, 저장소, 데이터베이스 등의 인프라를 필요할 때 빌려 쓰는 클라우드 서비스

---

## 3. 핵심 구성 요소

AWS는 아래 4가지 핵심 요소로 구성됩니다:

| 구분 | 서비스 | 역할 |
|------|--------|------|
| 서버 | EC2 | 애플리케이션 실행 |
| 저장소 | S3 | 파일 저장 |
| 데이터베이스 | RDS | 데이터 관리 |
| 권한 | IAM | 접근 제어 |

---

## 4. 전체 시스템 구조 이해

기본적인 AWS 아키텍처는 다음과 같습니다:

```
사용자 → EC2 → RDS
              ↓
             S3
```

### 동작 흐름
1. 사용자가 요청을 보냄  
2. EC2 서버가 요청 처리  
3. 데이터는 RDS에서 조회  
4. 파일은 S3에서 로드  

---

## 5. 단계별 학습 계획

### 📌 1단계: 기초 개념 및 서비스 이해 (1주)

#### 학습 내용
- AWS 계정 생성
- EC2 개념 이해 및 생성
- S3 구조 이해
- RDS 개념 이해
- IAM 권한 구조 이해

#### 목표
- AWS 기본 구조 이해
- 핵심 서비스 개념 숙지

---

### 📌 2단계: 기본 서비스 실습 (2~3주)

#### 실습 항목

1. **EC2 실습**
   - Linux 서버 생성
   - SSH 접속
   - 간단한 프로그램 실행

2. **S3 실습**
   - 버킷 생성
   - 파일 업로드 및 다운로드

3. **RDS 실습**
   - DB 생성
   - EC2에서 DB 연결

4. **IAM 실습**
   - 사용자 생성
   - 권한 정책 적용

#### 목표
- AWS 콘솔 사용 능력 확보
- 기본 인프라 구성 가능

---

### 📌 3단계: 네트워크 이해 (2주)

#### 학습 내용
- VPC
- Subnet (Public / Private)
- Internet Gateway
- NAT Gateway
- Security Group / NACL

#### 목표
- 네트워크 흐름 이해
- 외부/내부 접근 구조 설계 가능

---

### 📌 4단계: 아키텍처 설계 (2~3주)

#### 실습 프로젝트

1. **기본 웹 서버 구조**
   - EC2 + RDS + S3 구성

2. **로드밸런서 구조**
   - ALB 구성
   - 다중 서버 연결

3. **Auto Scaling**
   - 트래픽 기반 서버 자동 확장

#### 목표
- 고가용성 구조 이해
- 실무형 아키텍처 설계 능력 확보

---
